# E3039 Details

## Message emitted

`E3039: {Feature} {job|play} skipped — no devices in scope of any Golden Config Setting.`

or

`E3039: {Feature} {job|play} skipped — all {N} in-scope devices have it disabled on their winning Setting (see E3038 entries above).`

## Description

A summary line emitted once per feature when the job's filtered device set is empty. `{Feature}` is one of `Backup`, `Intended`, `Compliance`, `Plan`, or `Deploy`. The `{job|play}` token is `job` for single-feature jobs (`BackupJob`, `IntendedJob`, `ComplianceJob`, `GenerateConfigPlans`, `DeployConfigPlans`) and `play` for the per-feature emissions inside the all-in-one `Execute All Golden Configuration Jobs` jobs.

The first wording fires when no device in the job's scope matched any `Golden Config Setting`. The second wording fires when every in-scope device was already named in a preceding `E3038` entry — the line is a concise pointer back to those.

`E3039` is suppressed when exactly one device was filtered. The single `E3038` already names the device, so a one-device summary would just be noise.

## Troubleshooting

1. If the first wording fires, confirm a `Golden Config Setting` exists whose Dynamic Group matches at least one of the devices the job was asked to run against. Use the Setting's Dynamic Group preview to confirm membership.
2. If the second wording fires, scroll up to the preceding `E3038` entries — each names the specific Setting and weight that excluded the device. Resolve those (see [E3038](E3038.md)) and the `E3039` will stop firing.

## Recommendation

Either expand a `Golden Config Setting`'s Dynamic Group to include the intended devices, or enable the corresponding `enable_<feature>` flag on the winning Setting(s) named by the preceding `E3038` entries.
