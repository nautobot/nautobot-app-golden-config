# E3038 Details

## Message emitted

`E3038: Device {device.name} skipped for {feature} job — highest-weighted Golden Config Setting '{setting.name}' (weight {setting.weight}) has enable_{feature}=False.`

or

`E3038: Device {device.name} skipped for {feature} job — no eligible Golden Config Setting.`

## Description

The job did not run for this device because its highest-weighted `Golden Config Setting` has the corresponding `enable_<feature>` flag set to `False`. `<feature>` is one of `backup`, `intended`, `compliance`, `plan`, or `deploy`.

A device is routed to exactly one `Golden Config Setting` — the one with the highest `weight` whose Dynamic Group includes the device. If that winning Setting has the feature disabled, the job will not run for the device, even if a lower-weighted Setting in the device's scope has the feature enabled. The second wording (`no eligible Golden Config Setting`) means the device is not in scope of any Setting at all (this is most common for `Deploy Config Plans` when a plan exists for a device that was later moved out of every Setting's Dynamic Group).

## Troubleshooting

1. Open the named `Golden Config Setting` and confirm whether the `Enable {feature}` flag is intentionally off.
2. If two or more Settings have overlapping scope, list them ordered by `weight` (descending) — the highest-weighted one wins; the others are ignored for this device.
3. If you intended a lower-weighted Setting to win, either raise its weight above the named one or narrow the named Setting's Dynamic Group so it no longer matches this device.

## Recommendation

Enable the corresponding `enable_<feature>` flag on the winning `Golden Config Setting`, or adjust Setting weights / Dynamic Group scope so a different Setting wins for this device.
